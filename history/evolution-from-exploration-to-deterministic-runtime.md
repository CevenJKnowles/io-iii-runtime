# HIST-001 — Evolution from Exploration to Deterministic Runtime

This document records the high-level evolution of the IO project lineage:

- IO-I: exploratory phase (learning, experiments, early constraints)
- IO-II: persona/protocol phase (interaction discipline, consistency work)
- IO-III: deterministic runtime phase (portable reference implementation, explicit invariants)

Goal: preserve historical context without impacting the active runtime or creating ambiguity about what is executable.

Non-goals:
- No behavioral claims beyond what is in the repository history.
- No runtime changes; IO-III remains the sole active reference runtime surface.
