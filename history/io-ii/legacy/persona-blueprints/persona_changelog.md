This changelog tracks all meaningful changes to the **Io Persona** and related

## IO-II v.1.3.0 — Current Stable

- **Persona:** Io v1.3
- **Collaborative Protocol:** ICP v1.0
- **Status:** Stable, implementation-ready

### Added
- Canonical system prompt for Io v1.3.
- Traits, mode system, clarification protocol, verification protocol.
- FACT / GUESS / OPINION labelling convention.
- Triadic Summary sub-module.
- Dual Tutor + Collaborator role.
- Io–Ceven Collaborative Protocol (ICP v1.0) for interaction optimisation.

### Changed
- Strengthened drift prevention and output structuring versus earlier versions.
- Clarified safety and emotional handling behaviours.
- Clarified tool usage expectations and verification defaults.

### Removed
- Legacy softeners and overly hedged language patterns from earlier Io versions.
- Redundant empathy-balancing heuristics.

## IO-II v.1.2.x — Archived (Pre-Stable Persona)

> Details to be filled retroactively from earlier blueprints as needed.

- Earlier trait weighting and mode behaviour.
- Less explicit protocols for FACT / GUESS / OPINION.
- Less formalised collaboration protocol.

## IO-II v.1.4.0 — Planned (Streamlined Kernel, Future)

> Status: Planned / not yet implemented.

### Planned Features
- Refined and simplified core system prompt for faster adaptation.
- Explicit separation of:
  - Io-Studio (creation-heavy profile),
  - Io-Tutor (learning-heavy profile),
  - Io-Dev (implementation-heavy profile).
- Improved test harness integration for behaviour regression checks.
- Optional cross-model tuning blocks (OpenAI, Claude, Mistral, LLaMA profiles).

## IO-III — Future Local Persona

> Status: Concept phase (high-level only).

- Local, multi-LLM persona orchestrator.
- Io as a “conductor” for local models, tools, and sensors.
- Strong reliance on:
  - Io v1.3 / v1.4 persona kernel,
  - Extended collaboration protocols,
  - Structured ADRs and behaviour tests.
